Project 1: Movie Recommendation System

Iteration 1: Requirements

Overview

Vision Statement

Goal is to build a movie recommendation system that is based on member ratings, social media sentiments and searching algorithms. The purpose of this is to give a list of movies that are recommended by the system and allows users to log into the system and search or add movies with a rating. The recommendations are based on the rankings and reviews given by members, mining of social media data, critic reviews, and ratings from Netflix. The idea is to expose members to various movies both past, present and future to get them to watch and allow members to add and recommend movies not in the system. We can then mine or rank the most popular movies and allow members to communicate with each other via recommendations. Membership system is a simple form of social networking where similar members can be put into a circle or friendship group.

Project 2: Web based Stock Forecaster System

Iteration 1: Requirements

Overview

Vision Statement

The goal of this project is to be able to use Web services for stock-prediction. Each Web service (WS) will track different stocks and when queried, issue a forecast about the price movement for a given stock. The client module acts as facilitator which gathers information from multiple Web Services or independent experts and then combines their answers into a single recommendation. If there are conflicting reports, the client may repeat the process of querying and combining the answers until a suitable answer is found.

The target customers are individuals who are trading fairly frequently (a couple of times per week) but not very frequently (several times a day). The data gathered should be from a list of companies and should be used for developing the prediction model. The forecaster service does the following the data:

  • For a given stock, suggest an action (buy, sell, hold) for one stock at a time
  • Recommend a stock to but from all the stocks that are being tracked or from all in a given sector.

Project 3: New York Subway Route Planner

Iteration 1: Requirements

Overview

Vision Statement

Goal of this project is use the NYC MTA data feeds to enable users to plan their routes from one location to another. The application takes real time data from subway and railway lines and performs route suggestions for users who enter their starting location and their end location with times also given. The idea is to give the client a more simplistic presentation of the routes using a graph and also the quickest time to get to their destination based on the arrival time they want to that destination. If there is no shortest path based on the time given, the system will let the user know that it isn’t possible.

The application also gives users updates through a buzz feed about traffic delays, emergencies and closings. The application also will provide suggestions based on real time data based on the user’s route for delays, closings and emergencies.

Project 4: NFL Wide Receiver Forecaster

Iteration 1: Requirements

Overview

Vision Statement

The Goal of this project is to develop an application that is used to recommend wide receivers in college for an NFL team to daft. The idea is to get data on college receivers based on particular requirements that NFL scouts and Draft analysts use to judge how productive a college wide receiver will be in the NFL. The idea is to use machine learning in a way that it uses data of current and retired wide receivers as a method to project a college wide out. Based on the analysis a top 100 board of wide outs currently in college can be kept with the ability to mine each player’s data for further analysis. The data being tracked will follow the player from his freshman or red shirt year all the way to declaring from the NFL draft. The application will also use data from a players pro-day and combine results to make further analysis.

Project 5: Marvel Universe Search

Iteration 1: Requirements

Overview

Vision Statement

The goal is to create a web based search engine that allows users to query it to get information about the vast universe of Marvel characters, comics, movies and television shows. The engine is based mostly on Marvel Comics API and will be used to process queries and present the results to users. Due to the rate limit of calls to the API in a day, some form of caching is going to be used to try to limit the API calls and to improve performance.

Also data will be scraped from various other sources on the web to glean further insights. A list of top searches will be shown and current and future Marvel projects will be displayed for users to view for further information.
